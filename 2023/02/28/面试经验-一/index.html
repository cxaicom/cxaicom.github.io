1. Vue2和Vue3区别

  • options API和composition API
    Vue2中选项API(options API)格式的代码可读性较差，比如需要实现某一功能涉及到props、data、methods、computed、watch等属性，就会出现对应代码块，这样对相同数据进行操作使用的代码会被分割到各个属性中，不利于代码阅读
    Vue3的组合式API(composition API)就对这一缺点进行优化，使用组合式API我们能够将想要关联的代码放到一起，增加代码的可读性和可维护性。
  • 逻辑复用
    Vue2的选项API(options API),使我们的代码大量使用到this
    组件中的props,data,methods都是绑定this上下文，由this访问，不利于代码逻辑的复用
    Vue3使用组合式API涉及到跨组件之间提取、复用逻辑灵活

    打破this的限制，一个合成函数只依赖于它的参数和全局引入的Vue APIs，而不是this上下文。只需将组件中需要复用的代码抽离，将它导出为函数即可

Vue3对比Vue2优势

  1. Vue3使用composition API，让组件拥有了更加良好的代码组织结构；
  2. Vue3采用函数式编程方式，打破了this的限制，能够更好的复用性，真正体现实现功能的高内聚低耦合，更利于代码的可扩展性和可维护性。
  3. Vue3更友好的支持兼容TS

    响应式原理

    Vue2:
  4. Object.defineProperty只能劫持对象的属性,因此我们需要对每个对象的每个属性进行遍历。
  5. 在遇到一个对象的属性还是一个对象的情况下，需要递归监听，对性能影响比较大。
  6. 由于Vue会在初始化实例时对属性执行getter/setter转化，所有属性必须在data对象上存在才能让Vue将它转换为响应式。
  7. 对于已经创建的实例，Vue 不允许动态添加根级别的响应式属性。但是，可以使用 Vue.set(object, propertyName, value) 或vm.$set方法向嵌套对象添加响应式属性。(深入响应式原理)
  8. Object.defineProperty，不具备监听数组的能力，需要重新定义数组的原型来达到响应式。（变异数组方法）

Vue3：

  1. 基于Proxy和Reflect，可以原生监听数组，可以监听对象属性的添加和删除或多层嵌套数据结构的响应。
  2. Proxy代理的是整个对象，而不是对象的某个特定属性，不需要我们通过遍历来逐个进行数据绑定。
  3. 不需要一次性遍历data的属性，可以显著提高性能。
